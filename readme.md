# ShopApp

ShopApp — настольное приложение на .NET/Avalonia для просмотра и управления каталогом товаров.  
Решает задачу простого локального учета товаров: хранение описания, изображений, цен и скидок, отображение для гостя и простой административный интерфейс для CRUD-операций.

Что это делает: предоставляет удобный UI для просмотра товара (гость) и управления справочниками/товарами (админ). Устраняет необходимость держать каталог в разрозненных файлах — все данные хранятся в локальном хранилище (файловая БД/JSON/SQLite) и показываются в едином интерфейсе.

## Быстрый старт — требования
- Windows 10/11 (x64)
- .NET SDK/Runtime 6+ (установите с https://dotnet.microsoft.com/)
- Рекомендуется 4+ ГБ ОЗУ, 200 МБ свободного диска

## Установка и настройка
1. Клонировать репозиторий:
bash
git clone <repo-url> "c:\Users\brawl\OneDrive\Рабочий стол\ShopApp"
cd "c:\Users\brawl\OneDrive\Рабочий стол\ShopApp"
   
3. Восстановить зависимости и собрать:
bash
dotnet restore
dotnet build -c Debug
   
4. Настроить данные:
- Положить файлы изображений в папку Images/ (например Images/Icon.jpg).
- Поместить шаблон БД или JSON в папку data/ (например shop.db или products.json). При первом запуске приложение может инициализировать пустую БД.
4. Запустить приложение:
bash
dotnet run --project .
   
5. Подготовка дистрибутива (опционально):
- Публикация для Windows x64 (самодостаточный):
 bash
 dotnet publish -c Release -r win-x64 --self-contained true -o dist\win-x64
- Копировать папки data/ и Images/ вместе с опубликованными файлами.

## Конфигурация данных
- Формат импорта: CSV/JSON с полями: Name, Article, Category, Price, Discount, Unit, Quantity, ImagePath, Manufacturer, Supplier, Description.
- Путь к картинке в поле ImagePath может быть относительным (от папки приложения/Images) или абсолютным.
- При отсутствии изображения используется Images/Icon.jpg или заглушка "Нет фото".

## Использование
- Запустите EXE или через dotnet run — откроется главное окно.
Роли:
- Гость — просматривает каталог товаров, видит изображения, цену и скидки.
- Администратор — доступ к CRUD для товаров и справочников (если реализовано).
Обычные действия:
- Просмотр карточек товара в основном окне.
- При наличии скидки отображается зачеркнутая оригинальная цена и финальная цена.
- Для администрирования используйте админ-окно (Login/Role в приложении).
